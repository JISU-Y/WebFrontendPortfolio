when & where

프로젝트 설명:
Vanilla JS Web Site 개인 프로젝트

세계 여행에 필요한 나라별/월별 기후를 알려주는 Web Site
worldbank - Climate Data API 사용
검색창에 국가명(영어 기준)을 검색, 자동 추천 기능
월을 선택하여 선택된 국가의 해당 월의 기후(평균 기온, 강수량, 일교차)를 그에 맞는 이미지와 함께 확인 가능
모든 기능은 plugin없이 vanilla JS로 구현 어려웠던 점:
처음 API를 사용하면서 제공하는 데이터 이용방법이 매우 어려웠다.
INPUT에서 국가 검색 기능을 추가하는 것이 어려웠다.
기능을 먼저 생각하고 구현하다보니 페이지 디자인이 떨어지게 되었다.

배운점:
잘 사용하지 않는 API를 사용했지만, 검색해보며 API 사용법은 제공 사이트에 방법이 잘 설명되어 있다는 것을 알게되어 구현할 수 있었다.
라이브러리없이 국가 정보 API를 통해 국가 검색 기능을 구현할 수 있게 되었다.
UI를 더욱 깔끔하게 하기 위해 tailwind로 리팩토링하여 사용방법 및 웹 사이트 디자인에 도움이 되었다.

Post It!

프로젝트 설명:
Full Stack(MERN Stack) 개인 프로젝트

Post it처럼 원하는 곳에 붙일 수 있는 Web App
Post 속성(Today/Later/Important..) 선택 설정 가능
Post내 to do 목록 complete, delete, update 가능
Google Login 및 JWT Login 구현
Login User만 Post it 생성/수정/삭제 가능(Auth)
Express와 Mongoose로 Backend 및 DB 구축 어려웠던 점:
리팩토링 과정에서 React, Express, Mongoose 등 새로운 기술의 학습이 계속해서 필요했다.
사용자 친화적이고 직관적인 UX를 구현하는 것이 어려웠다.

배운점:
React에서 useState를 이용한 상태 관리와 props 전달하여 상태를 관리하는 방식을 익힐 수 있었다.
Mongoose에서 POST의 속성을 저장하기 위한 간단한 Schema를 작성할 수 있게 되었다.
Express를 통해 CRUD 기능을 구현하여 Server단을 구성할 수 있었다.
코드의 가독성을 위해 Component를 나누고, 상태 관리를 위해 Redux의 필요성을 알게 되었다.
React Express(Node.js) Mongoose(MongoDB)
